IT 개발에서는 GIT은 필수입니다.
GIT을 모르거나 안쓰면 작업이 진행이 안될 정도입니다.

예시:
A라는 사람이 python_a.py에 작업을 함
B라는 사람이 python_b.py,python_c.py 에 작업을 함

이것을 취합한다고 할시 git을 안쓰면 대 혼란이 일어남

옛날에는 USB 사용.
이럴때의 문제점은
A라는 사람은 list1.py 파일에 코드를 추가하고 삭제하고 변경했고
B라는 사람은 그걸 모르고 list.py에 코드 추가하고 있었음
USB 파일을 덮어 씌워버리면 특정 사람이 한 코드는 다 날라감

GIT은 진짜 강력하지만 사용법이 어려움
내가 만든 프로젝트 올리고, 최신코드 반영하는 정도로 익혀둠
진짜 깊숙히 알아야한다-
branch 나누는법, 
git action

GIT이 어려운 이유2-
pull 혹은 MERGE 할때 에러로그뜸

GIT을 쓸때 습관-
항상 작업하기전에 pull 먼저 해야함

GIT 기본 사용 순서
pull - git 서버로부터 최신코드 가져옴
commit - 내가 작업한거 저장
pull
push
